## 💡 주요 학습 내용

### 1️⃣ 기술적 핵심 (Technical Points)
어제 구축한 서버 아키텍처가 실제로 클라이언트와 어떻게 소통하는지, 그 '대화 과정(HTTP)'을 정밀하게 분석하고 기록하는 법을 학습했습니다.

* **HTTP 요청-응답 구조 시각화 (Excalidraw)**
    * 클라이언트와 서버 간의 데이터 흐름을 시각화하며 `Method`, `URL`, `Header`, `Body`, `Status Code`의 연관 관계를 정리했습니다.
    * RESTful API 설계의 기본이 되는 HTTP 전송 방식의 원리를 파악했습니다.

* **실시간 API 로깅 (Loguru & Middleware)**
    * 단순 `print`가 아닌 전문 로깅 라이브러리 `Loguru`를 도입했습니다.
    * 서버에 들어오는 모든 요청의 상세 정보(Header, Body, IP 등)와 서버의 응답 상태를 콘솔에 실시간으로 출력하여 디버깅 효율을 높였습니다.

* **응답 포맷 분석 (JSON vs HTML)**
    * 동일한 HTTP 통신이라도 목적에 따라 `Content-Type`이 어떻게 달라지는지 비교 실습했습니다.
    * 기계 간 데이터 통신을 위한 **JSON**과 사용자 화면 출력을 위한 **HTML**의 구조적 차이와 MIME 타입 개념을 정리했습니다.

### 2️⃣ 논의 및 회고 (Discussion & Reflection)
* **핵심 깨달음**: "어제 배운 ORM과 아키텍처가 서버의 '내장'이라면, 오늘 배운 HTTP와 로깅은 서버의 '입과 귀'이다. 잘 들리고 잘 말해야(Logging) 어디가 문제인지(Error) 금방 찾을 수 있다."
* **느낀 점**: 어제 어려운 개념들을 미리 잡아둔 덕분에, 오늘은 네트워크의 흐름을 한결 여유롭게 조망할 수 있었습니다. 기본기가 탄탄해야 복잡한 아키텍처도 의미가 있다는 것을 깨달았습니다.
